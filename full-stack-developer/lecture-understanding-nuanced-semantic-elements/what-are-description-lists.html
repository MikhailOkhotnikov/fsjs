Что такое списки описаний и когда следует их использовать?

Списки описаний(description lists) идеально подходят для представления терминов и определений в организованном и удобном для чтения формате, как в глоссарии или настоящем словаре, где вы можете найти слова с соответствующими им определениями.

Это пример списка описаний в HTML с двумя терминами и соответствующими им данными: 

    
      <dl>
        <dt>HTML</dt>
        <dd>Язык гипертекстовой разметки</dd>
        <dt>CSS</dt>
        <dd>Каскадные таблицы стилей</dd>
      </dl>

В данном случае термины — это аббревиатуры HTML и CSS, а подробности — их расшифровки. Подробности также могут быть определениями или другой информацией, связанной с терминами.

Для определения списка описаний вам понадобятся три HTML-элемента. Во-первых, элемент списка описаний, dl(description list), который является контейнером для всего списка. Как видите, в примере он обволакивает все остальные элементы списка описаний.

Затем один элемент описания термина, dt(description term), для каждого термина. В данном случае список описаний содержит два термина, HTML и CSS, поэтому в нём два таких элемента.

И наконец, после каждого термина вы увидите элемент описания, dd(description details), содержащий описание или сведения, связанные с этим термином. В данном примере это язык гипертекстовой разметки и каскадные таблицы стилей.

В браузере каждый термин отображается вместе с соответствующим описанием. По умолчанию описания имеют небольшой отступ вправо для визуального различения.

Но списки описаний не ограничиваются только терминами и определениями. Они гораздо более универсальны. Вот рецепт из двух ингредиентов. 

    
      <dl>
        <dt>Мука</dt>
        <dd>2 стакана</dd>
        <dt>Сахар</dt>
        <dd>1/2 стакана</dd>
      </dl>

Весь список описаний находится в элементе dl. Первый ингредиент, Мука, находится в элементе dt. Затем вы можете увидеть, сколько этого ингредиента вам понадобится: 2 стакана. Этот элемент находится в элементе dd сразу после соответствующего ингредиента.

Та же структура повторяется и для Сахар. В данном случае в рецепте всего два ингредиента, но если бы их было больше, та же структура могла бы повторяться во всём списке описания.

В браузере вы увидите ингредиенты, выровненные по левому краю, а количество с отступом, чтобы визуально разделить их.

Списки описаний также можно использовать для описания характеристик продукта, часто задаваемых вопросов, контактной информации и метаданных. По сути, когда у вас есть два связанных фрагмента информации в формате пары «ключ-значение», где один выступает в качестве метки (ключа), а другой — в качестве дополнительной связанной информации (значения), вы можете использовать список описаний.
