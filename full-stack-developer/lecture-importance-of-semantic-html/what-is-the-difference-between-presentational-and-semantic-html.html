В чем разница между презентационным и семантическим HTML?

Презентационный HTML фокусируется на внешнем виде и стиле контента. На заре развития HTML разработчики использовали такие элементы, как center, big и font. Но в современной веб-разработке такие элементы не следует использовать из-за их ограничений и негативного влияния на доступность и удобство поддержки.

Многие элементы оформления HTML устарели, то есть больше не рекомендуются к использованию. Существуют более эффективные способы добиться тех же результатов. Однако полезно знать, что они существуют, поэтому давайте рассмотрим несколько примеров.

Элемент font — устаревший элемент, используемый для задания размера и цвета шрифта текста. Вот пример элемента font: 

    
      <font size="5" color="blue">Этот текст выделен синим цветом и выделен крупным шрифтом.</font>

В этом примере цвет текста задаётся blue, а размер — значением 5. Диапазон значений атрибута size — от 1 до 7, где 1 — наименьшее значение, а 7 — наибольшее. Значение по умолчанию — 3, если вы не укажете значение явно.

Хотя этот элемент все еще работает, его не следует использовать, поскольку размер и цвет шрифта всегда должны задаваться в CSS, а не в HTML.

Элемент center — ещё один устаревший элемент, который используется для горизонтального центрирования содержимого внутри контейнера. Вот пример элемента center, содержащего текст и элемент абзаца: 

    
      <center>
        Этот текст расположен по центру.
        <p>HTML - это потрясающе.</p>
      </center>

В браузере вы увидите весь контент внутри элемента center, отцентрированный по горизонтали.

Далее идёт элемент big. Это ещё один устаревший презентационный HTML-элемент, который делает вложенный текст на один уровень больше окружающего его текста. Вот пример, определяющий абзац из двух частей: 

    
      <p>
        Этот текст имеет обычный размер шрифта.
        <big>Этот текст увеличен в размере.</big>
      </p>

Первая часть содержит текст обычного размера, а вторая часть, заключённая в элемент big, будет отображаться более крупным шрифтом. В браузере вы увидите, что текст внутри элемента big выглядит крупнее окружающего текста.

Однако помните, что размер шрифта всегда следует задавать с помощью CSS, поэтому не следует использовать этот элемент в современном HTML.

Это были примеры презентационных HTML-элементов. Но все они устарели и больше не рекомендуются. Так что же использовать вместо них? Давайте посмотрим.

Семантический HTML теперь является рекомендуемой практикой. Он описывает содержимое элементов, поэтому его гораздо легче читать, понимать и поддерживать.

Поисковые системы могут легко распознавать ваш сайт, используя семантический HTML. Это также полезно для обеспечения доступности, поскольку программам чтения с экрана необходима семантическая информация для описания содержимого веб-страницы.

Примеры семантических элементов HTML включают в себя:

  • Элемент header для определения заголовка документа или раздела.
  • Элемент раздела навигации, nav, для разделов с навигационными ссылками.
  • Элемент section для группировки связанной информации.
  • Элемент figure для иллюстраций и диаграмм.

Это пример элемента заголовка(header), содержащего элемент раздела навигации(nav): 

    
      <header>
        <nav>
          <a href="#">Главная страница</a>
          <a href="#">Информация</a>
          <a href="#">Контакты</a>
        </nav>
      </header>

Семантические элементы чётко отражают своё назначение в структуре HTML. Существует множество различных семантических элементов HTML. Вы обязательно найдёте тот, который соответствует потребностям вашего проекта.

Отличная работа. Теперь вы знаете разницу между презентационным и семантическим HTML: семантический HTML описывает содержание, а презентационный HTML фокусируется на внешнем виде.
