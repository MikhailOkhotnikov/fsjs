Для чего используются HTML-таблицы, и для чего их использовать не следует?

В наши дни HTML-таблицы используются не так часто, как раньше. Но, будучи фронтенд-разработчиком, вы всё равно должны быть с ними знакомы. Таблицы — один из первых способов отображения данных в браузере, появившихся ещё в 1990-х годах.

Вот пример кода, использованного для генерации таблицы из Бюро статистики труда США:

Пример кода 
      
        <table id="quickfacts">
          <thead>
            <tr>
              <th colspan="2">Краткие сведения: Разработчики программного обеспечения, аналитики по обеспечению качества и тестировщики</th>
            </tr>
          </thead>
          <tbody>
            <tr>
              <th>Средняя заработная плата в 2023 году</th>
              <td>
                130 160 долларов в год
                <br>62,58 доллара в час
              </td>
            </tr>
            <tr>
              <th>Типичное образование начального уровня</th>
              <td>Степень бакалавра</td>
            </tr>
            <tr>
              <th>Опыт работы по смежной специальности</th>
              <td>Отсутствует</td>
            </tr>
            <tr>
              <th>Обучение на рабочем месте</th>
              <td>Отсутствует</td>
            </tr>
            <tr>
              <th>Количество рабочих мест, 2022</th>
              <td>1,795,300</td>
            </tr>
            <tr>
              <th>Перспективы трудоустройства на 2022-32 годы</th>
              <td>25% (намного быстрее, чем в среднем)</td>
            </tr>
            <tr>
              <th>Изменение условий занятости, 2022-32 годы</th>
              <td>451,200</td>
            </tr>
          </tbody>
          <tfoot>
            <tr>
              <th>Если бы у этой таблицы был нижний колонтитул, он был бы здесь.</th>
            </tr>
          </tfoot>
        </table>

Как видите, здесь есть основной элемент table с id равным quickfacts. Внутри него находится элемент заголовка таблицы (thead, table head), элемент тела таблицы (tbody, table body) и элемент нижней части таблицы (tfoot, table foot).

Элементы заголовка, тела и нижнего колонтитула таблицы могут содержать некоторое количество строк таблицы (tr, table rows). Каждая строка таблицы может содержать заголовок таблицы (th, table header), который обозначает данные в этой строке. Она также может содержать некоторое количество отдельных ячеек данных, называемых данными таблицы (td, table data).

Итак, это множество HTML-элементов. Но не пугайтесь – они подчиняются простой иерархии.

Вот самая простая таблица, которую мы можем создать, включающая все эти элементы:

Пример кода 
      
        <table>
          <thead>
            <tr>
              <th>Заголовок этой таблицы</th>
            </tr>
          </thead>
          <tbody>
            <tr>
              <th>Первая строка</th>
              <td>
                Первая ячейка данных
              </td>
            </tr>
            <tr>
              <th>Вторая строка</th>
              <td>
                Вторая ячейка данных
              </td>
            </tr>
          </tbody>
          <tfoot>
            <tr>
              <th>В нижней части этой таблицы может содержаться дата публикации, информация об авторах или другие метаданные.</th>
            </tr>
          </tfoot>
        </table>

Как видите, сами данные всегда находятся внутри элемента tr, а внутри этого элемента tr находится элемент th с заголовком и элемент td с данными.

На некоторых сайтах для создания таблиц предпочитают использовать элементы div вместо более подходящего элемента table.

Хотя табличные данные можно отображать с помощью обычных элементов div, всё же лучше использовать элемент table.

Много лет назад разработчики могли использовать table для размещения не содержащих данных элементов на веб-странице. Это никогда не считалось лучшей практикой. Но вы можете встретить некоторые кодовые базы, где таблицы до сих пор используются подобным образом.

В настоящее время разработчики используют CSS flexbox и grid для компоновки своих дизайнов. Мы подробно рассмотрим эти инструменты позже.
