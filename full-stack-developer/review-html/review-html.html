Обзор HTML

Ознакомьтесь с приведенными ниже понятиями, чтобы подготовиться к предстоящему подготовительному экзамену.

Основы HTML

  • Роль HTML: HTML (Hypertext Markup Language) — это основа структуры веб-сайта, определяющая элементы веб-страницы.
  • HTML-элементы: используются для представления содержимого на странице. Большинство из них создаются с помощью открывающего и закрывающего тега (например, <h1></h1>, <p></p>).
  • Структура HTML: HTML состоит из заголовка (head) и тела (body), где структурируются метаданные, стили и контент.
  • Основные элементы HTML: заголовки h1 - h6, абзацы p и контейнеры div.
  • Элемент div: Элемент div — это универсальный HTML-элемент, не имеющий семантического значения. Он используется в качестве универсального контейнера для размещения других HTML-элементов.
  • Пустые элементы: не имеют закрывающего тега (например, <img>).
  • Атрибуты: добавление метаданных и поведения к элементам.

Идентификаторы и группировка

  • Идентификаторы id: Уникальные идентификаторы элементов.
  • Классы class: Группировка элементов для оформления и управления поведением.

Специальные символы и ссылки

  • HTML-сущности: Использование специальных символов, таких как &amp; и &lt;.
  • Элемент ссылки(link): Ссылка на внешние таблицы стилей.
  • Элемент скрипта(script): Встраивание внешних файлов JavaScript.

Шаблон и кодирование

  • HTML-шаблон: базовая структура веб-страницы, включающая элементы DOCTYPE, html, head и body. Он должен использоваться в качестве отправной точки для HTML-документа.
  • Кодировка символов UTF-8: Обеспечение универсального отображения символов.

SEO и обмен информацией в социальных сетях

  • Метатеги (description): предоставляют краткое описание веб-страницы и влияют на SEO.
  • Теги Open Graph: Улучшение обмена информацией в социальных сетях.

Медиаэлементы и оптимизация

  • Заменяемые элементы: Встроенный контент (например, изображения, iframe).
  • Оптимизация медиаконтента: методы повышения эффективности медиаресурсов.
  • Форматы и лицензии изображений: понимание прав и типов использования.
  • SVG: Масштабируемая векторная графика для четких изображений.

Интеграция мультимедиа

  • HTML audio и video элементы: встраивание мультимедиа.
  • Встраивание с помощью iframe: интеграция внешнего видеоконтента.

Пути и поведение ссылок

  • Целевые типы атрибутов: Управление поведением ссылок.
  • Абсолютные и относительные пути: навигация по каталогам.
  • Синтаксис путей: понимание /, ./, ../ для навигации по файлам.
  • Состояния ссылок: Управление различными взаимодействиями со ссылками (при наведении курсора(hover), в активном состоянии(active)).
  • Внутренние ссылки: создание ссылки на другой раздел страницы с помощью атрибута href="#id" для элемента a и присвоения целевому элементу того же идентификатора(id).

Важность семантического HTML

  • Структурная иерархия элементов заголовков: Важно использовать правильный элемент заголовка для поддержания структурной иерархии контента. Элемент h1 — это самый верхний уровень заголовка, а элемент h6 — самый нижний.
  • Презентационные HTML элементы: элементы, которые задают его внешний вид, например, устаревшие элементы center, big и font.
  • Семантические HTML элементы: Эти элементы придают смысл структуре контента. Примеры:
    • header: Представляет собой вводный контент.
    • nav: Содержит навигационные ссылки.
    • article: Представляет собой самодостаточный контент.
    • aside: Используется для боковых панелей или связанного контента.
    • section: Группирует связанный контент внутри документа.
    • footer: Определяет нижний колонтитул раздела или документа.

Семантические HTML элементы

  • Элемент выделения (em, emphasis): обозначает текст, в котором выделены смысловые ударения.
  • Элемент идиоматического текста (i, idiomatic): Используется для выделения альтернативного стиля или настроения, идиоматических выражений из другого языка, технических терминов и мыслей.
  • Элемент «Высокая важность»(strong): обозначает текст, имеющий высокую важность.
  • Привлечь внимание к элементу (b, bring): Используется для привлечения внимания к тексту, который не важен для смысла содержания.
  • Элемент список описаний (dl, Description List): используется для представления списка групп терминов и описаний.
  • Элемент описания термина (dt, Description Term): используется для обозначения определяемого термина.
  • Элемент Описание и подробности (dd, Description Details): используется для представления описания термина.
  • Элемент Блочной цитаты (blockquote): используется для обозначения раздела, цитируемого из другого источника.
  • Элемент Встроенной цитата (q, Quotation): Используется для обозначения короткой цитаты, размещенной непосредственно в тексте.
  • Элемент Сокращение (abbr, Abbreviation): Используется для обозначения аббревиатуры или акронима.
  • Элемент «Контактный адрес» (address): используется для представления контактной информации.
  • (Дата) Время (time): Используется для обозначения даты и/или времени.
  • Верхний (надстрочный) индекс (sup, superscript): Используется для обозначения текста в верхнем индексе.
  • Нижний (подстрочный) элемент(sub, subscript): используется для представления текста с нижним индексом.
  • Встроенный элемент кода (code): используется для представления фрагмента компьютерного кода.
  • Элемент Неформализованная аннотация (u, unarticulated annotation): Используется для обозначения фрагмента встроенного текста, который должен отображаться таким образом, чтобы указывать на наличие нетекстовой аннотации.
  • Элемент Аннотация Ruby (ruby): предназначен для добавления небольшой аннотации сверху или снизу от заданного текста. Обычно используется для указания произношения восточноазиатских символов (например, китайского и японского).
  • Элемент "Зачеркнуто" (s, strikethrough): используется для обозначения контента, который больше не является актуальным или точным.

Элементы и атрибуты HTML-форм

Формы

  • Элемент form: Используется для создания HTML-формы для ввода данных пользователем.
  • Типичные типы входных данных:
    • text, email, password, radio, checkbox, number, date.
  • Атрибут action: используется для указания URL-адреса, куда должны быть отправлены данные формы.
  • Атрибут method: используется для указания метода HTTP, который будет применяться при отправке данных формы. Наиболее распространенные методы — GET и POST. 
    •       
        <form method="значение находится здесь" action="url-адрес находится здесь">
          <!-- Здесь размещаются входные данные. -->
        </form>
  • Элемент input: используется для создания поля ввода для ввода данных пользователем.
  • Атрибут type: используется для указания типа поля ввода, например, text, email, number, radio, checkbox.
  • Атрибут placeholder: используется для отображения подсказки пользователю, показывающей, что нужно ввести в поле ввода.
  • Атрибут value: используется для указания значения поля ввода. Если поле ввода имеет тип button, атрибут value можно использовать для установки текста кнопки.
  • Атрибут name: используется для присвоения имени полю ввода, которое служит ключом при отправке данных формы. Для переключателей (радиокнопок) присвоение им одинакового имени группирует их вместе, так что одновременно можно выбрать только один вариант из группы.
  • Атрибут size: используется для определения количества символов, которые должны быть видны, когда пользователь вводит текст в поле ввода.
  • Атрибут min: может использоваться с типами ввода, например, number, для указания минимального допустимого значения в поле ввода.
  • Атрибут max: может использоваться с типами ввода, такими как number, для указания максимально допустимого значения в поле ввода.
  • Атрибут minlength: используется для указания минимального количества символов, необходимых для заполнения поля ввода.
  • Атрибут maxlength: используется для указания максимального количества символов, разрешенных в поле ввода.
  • Атрибут required: используется для указания того, что поле ввода обязательно должно быть заполнено перед отправкой формы.
  • Атрибут disabled: используется для указания того, что поле ввода должно быть отключено.
  • Атрибут readonly: используется для указания того, что поле ввода доступно только для чтения. 
    •       
        <!-- Текстовое поле ввода -->
        <input 
          type="text"
          id="name"
          name="name"
          placeholder="e.g. Quincy Larson"
          size="20"
          minlength="5"
          maxlength="30"
          required
        />

        <!-- Числовое поле ввода -->
        <input 
          type="number"
          id="quantity"
          name="quantity"
          min="2"
          max="10"
          disabled
        />

        <!-- Кнопка-->
        <input type="button" value="Показать оповещение" />
  • Элемент label: используется для создания метки для поля ввода.
  • Атрибут for: используется для указания того, к какому полю ввода относится метка.
  • Неявная привязка формы: поля ввода можно связать с метками, заключив поле ввода в элемент метки. 
    •       
        <form action="">
          <label>
            ФИО:
            <input type="text" />
          </label>
        </form>
  • Явная привязка к форме: поля ввода можно связать с метками, используя атрибут for элемента label. 
    •       
        <form action="">
          <label for="email">Адрес электронной почты:</label>
          <input type="email" id="email" />
        </form>
  • Элемент button: используется для создания кликабельной кнопки. Кнопка также может иметь атрибут type, который управляет поведением кнопки при активации, например, submit, reset, button. 
    •       
        <button type="button">Показать форму</button>
        <button type="submit">Отправить форму</button>
        <button type="reset">Сбросить форму</button>
  • Элемент fieldset: используется для группировки связанных полей ввода.
  • Элемент legend: используется для добавления подписи, описывающей группу входных данных. 
    •       
        <!-- Радиогруппа -->
        <fieldset>
          <legend>Вы впервые остановились в нашем отеле?</legend>

          <input id="yes-option" type="radio" name="hotel-stay" value="yes" />
          <label for="yes-option">Да</label>

          <input id="no-option" type="radio" name="hotel-stay" value="no" />
          <label for="no-option">Нет</label>
        </fieldset>

        <!-- Группа флажков -->
        <fieldset>
          <legend>
            Почему вы выбрали именно наш отель для проживания? (Отметьте все подходящие варианты)
          </legend>

          <input type="checkbox" id="location" name="location" value="location" />
          <label for="location">Расположение</label>

          <input type="checkbox" id="price" name="price" value="price" />
          <label for="price">Цена</label>
        </fieldset>
  • Состояние фокусировки: это состояние поля ввода, когда оно выбрано пользователем.

Работа с элементами и атрибутами HTML-таблиц

  • Элемент table: используется для создания HTML-таблицы.
  • Элемент Table Head (thead): используется для группировки содержимого заголовка в HTML-таблице.
  • Элемент Table Row (tr): используется для создания строки в HTML-таблице.
  • Элемент Table Header (th): используется для создания ячейки заголовка в HTML-таблице.
  • Элемент Table body (tbody): используется для группировки основного содержимого в HTML-таблице.
  • Элемент ячейки Table Data (td): используется для создания ячейки данных в HTML-таблице.
  • Элемент Table Foot (tfoot): используется для группировки содержимого нижнего колонтитула в HTML-таблице.
  • Элемент caption: используется для добавления заголовка HTML-таблицы.
  • Атрибут colspan: используется для указания количества столбцов, которые должна занимать ячейка таблицы. 
    •       
        <table>
          <caption>Оценки за итоговый экзамен по математическому анализу</caption>
          <thead>
            <tr>
              <th>Фамилия</th>
              <th>Имя</th>
              <th>Оценка</th>
            </tr>
          </thead>
          <tbody>
            <tr>
              <td>Иванов</td>
              <td>Иван</td>
              <td>5</td>
            </tr>
            <tr>
              <td>Петров</td>
              <td>Пётр</td>
              <td>4</td>
            </tr>
            <tr>
              <td>Сидоров</td>
              <td>Сидор</td>
              <td>3</td>
            </tr>
            <tr>
              <td>Михайлов</td>
              <td>Михаил</td>
              <td>4</td>
            </tr>
            <tr>
              <td>Дмитриев</td>
              <td>Дмитрий</td>
              <td>5</td>
            </tr>     
          </tbody>
          <tfoot>
            <tr>
              <td colspan="2">Средняя оценка</td>      
              <td>4,2</td>
            </tr>
          </tfoot>
        </table>

HTML инструменты

  • Валидатор HTML: инструмент, проверяющий синтаксис HTML-кода на предмет его корректности.
  • Инспектор DOM: инструмент, позволяющий проверять и изменять HTML-структуру веб-страницы.
  • Devtools: набор инструментов для веб-разработчиков, встроенных непосредственно в браузер, которые помогают отлаживать, профилировать и анализировать веб-страницы.

Введение в доступность

  • Рекомендации по доступности веб-контента: Рекомендации по доступности веб-контента (WCAG) — это набор руководящих принципов, призванных сделать веб-контент более доступным для людей с ограниченными возможностями. Четыре принципа WCAG — это POUR, что означает Perceivable(воспринимаемый), Operable(управляемый), Understandable(понятный), Robust(надежный).

Вспомогательные технологии для обеспечения доступности

  • Программы для чтения с экрана: Программные средства, которые озвучивают содержимое экрана компьютера. Они используются людьми с нарушениями зрения для доступа к интернету.
  • Клавиатуры с крупным текстом или брайлевские клавиатуры: используются людьми с нарушениями зрения для доступа к интернету.
  • Экранные увеличители: Программные средства, увеличивающие содержимое экрана компьютера. Используются людьми со слабым зрением для доступа к интернету.
  • Альтернативные указывающие устройства: используются людьми с ограниченными возможностями передвижения для доступа к интернету. К ним относятся такие устройства, как джойстики, трекболы и сенсорные панели.
  • Распознавание голоса: используется людьми с ограниченными возможностями передвижения для доступа к интернету. Оно позволяет пользователям управлять компьютером с помощью голоса.

Инструменты аудита доступности

  • Распространенные инструменты проверки доступности: Google Lighthouse, Wave, IBM Equal Accessibility Checker и axe DevTools — это некоторые из распространенных инструментов, используемых для проверки доступности веб-сайта.

Лучшие практики по обеспечению доступности

  • Правильная структура заголовков: следует использовать правильную структуру заголовков для создания логического представления контента. Это помогает вспомогательным технологиям понимать содержимое вашего веб-сайта.
  • Доступность и таблицы: При использовании таблиц следует использовать элемент th для определения ячеек заголовка и элемент td для определения ячеек данных. Это помогает вспомогательным технологиям понимать структуру таблицы.
  • Важность наличия связанной метки для полей ввода: следует использовать элемент label для связывания меток с полями ввода формы. Это помогает вспомогательным технологиям понимать назначение поля ввода.
  • Важность качественного альтернативного текста: Атрибут alt следует использовать для предоставления текстовой альтернативы изображениям. Это помогает вспомогательным технологиям понимать содержимое изображения.
  • Важность качественного текста ссылки: следует использовать описательный текст ссылки, чтобы помочь пользователям понять назначение ссылки.
  • Лучшие практики по обеспечению доступности аудио и видео контента: Для людей с нарушениями слуха следует предоставлять субтитры и расшифровки аудио и видео контента. Для людей с нарушениями зрения следует также предоставлять аудиоописания видеоконтента.
  • Атрибут tabindex: используется для того, чтобы сделать элементы фокусируемыми и определить относительный порядок навигации по ним с помощью клавиатуры. Важно никогда не использовать атрибут tabindex со значением больше 0. Вместо этого следует использовать значение 0 или -1.
  • Атрибут accesskey: используется для определения сочетания клавиш для элемента. Это может помочь пользователям с ограниченными возможностями передвижения легче перемещаться по веб-сайту.

WAI-ARIA, роли и атрибуты

  • WAI-ARIA: это аббревиатура от Web Accessibility Initiative - Accessible Rich Internet Applications (Инициатива по обеспечению доступности веб-сайтов — доступные многофункциональные интернет-приложения). Это набор атрибутов, которые можно добавлять к элементам HTML для повышения доступности. Он предоставляет вспомогательным технологиям дополнительную информацию о назначении и структуре контента.
  • Роли ARIA: набор предопределенных ролей, которые можно добавить к элементам HTML для определения их назначения. Это помогает вспомогательным технологиям понимать содержимое веб-сайта. Примеры включают role="tab", role="menu" и role="alert".
  • Атрибуты aria-label и aria-labelledby: Эти атрибуты используются для присвоения элементу программного (или доступного) имени, что помогает вспомогательным технологиям (например, программам чтения с экрана) понимать назначение элемента. Они часто используются, когда визуальная метка элемента представляет собой изображение или символ, а не текст. Атрибут aria-label позволяет определить имя непосредственно в атрибуте, а aria-labelledby позволяет ссылаться на существующий текст на странице.
  • Атрибут aria-hidden: используется для скрытия элемента от вспомогательных технологий, таких как программы чтения с экрана. Например, его можно использовать для скрытия декоративных изображений, не содержащих никакого осмысленного контента.
  • Атрибут aria-describedby: Используется для предоставления дополнительной информации об элементе путем связывания его с другим элементом, содержащим эту информацию. Это помогает вспомогательным технологиям понять назначение элемента.
