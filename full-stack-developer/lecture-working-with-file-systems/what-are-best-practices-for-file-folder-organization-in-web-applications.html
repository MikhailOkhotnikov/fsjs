Каковы лучшие практики организации файлов/папок в веб-приложениях?

При разработке веб-приложений важно продумать организацию файлов и папок. Это поможет поддерживать код в упорядоченном виде и упростит его сопровождение.

Давайте рассмотрим пример структуры папок для проекта, использующего HTML и CSS: 

    
      .
      ├── /assets
      │   ├── /images
      │   │   ├── logo.png
      │   │   ├── banner.jpg
      │   │   └── icons.svg
      │   ├── /fonts
      │   │   ├── custom-font.woff
      │   │   └── custom-font.woff2
      ├── /css
      │   ├── main.css
      │   ├── about.css
      │   └── contact.css
      ├── index.html
      ├── about.html
      ├── contact.html
      └── README.md

В верхней части примера находится одна точка, обозначающая корневой каталог.

Корневой каталог — это каталог верхнего уровня в файловой системе, служащий отправной точкой для всех путей к файлам и содержащий все остальные каталоги и файлы. Каталог — это другое название папки.

Внутри корневого каталога первым отображается каталог assets. Термин assets (ресурсы) обозначает любые файлы, используемые в проекте, такие как изображения, шрифты или другие ресурсы.

В этом примере внутри каталога assets находятся две папки: images и fonts.

Следующая папка — это каталог css. Иногда её также называют директорией таблиц стилей. Здесь вы будете хранить все свои CSS-файлы.

В каталоге css находятся HTML-файлы. Файл index.html представляет собой главную страницу, а файлы about.html и contact.html — страницы «О нас» и «Контакты» соответственно.

Наконец, есть файл README.md. Файлы README обычно используются для предоставления полезной информации о проекте, например, о том, что он делает, как его использовать и любые другие важные детали.

Расширение .md означает Markdown, это легковесный язык разметки, часто используемый для написания документации.

Разделение проекта на файлы assets, CSS и HTML упрощает поиск нужной информации. Это также облегчает другим разработчикам внесение своего вклада в ваш проект.

Важно отметить, что это лишь один из примеров организации файлов и папок. Существует множество различных способов структурирования проекта.
