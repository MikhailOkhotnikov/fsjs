Обзор доступности HTML

Введение в доступность

  • Рекомендации по доступности веб-контента: Рекомендации по доступности веб-контента (Web Content Accessibility Guidelines, WCAG) — это набор руководящих принципов, призванных сделать веб-контент более доступным для людей с ограниченными возможностями. Четыре принципа WCAG — это POUR, что означает «воспринимаемый»(Perceivable), «управляемый»(Operable), «понятный»(Understandable) и «надежный»(Robust).

Вспомогательные технологии для обеспечения доступности

  • Программы для чтения с экрана: Программные средства, которые озвучивают содержимое экрана компьютера. Они используются людьми с нарушениями зрения для доступа к интернету.
  • Клавиатуры с крупным текстом или брайлевские клавиатуры: используются людьми с нарушениями зрения для доступа к интернету.
  • Экранные увеличители: Программные средства, увеличивающие содержимое экрана компьютера. Используются людьми со слабым зрением для доступа к интернету.
  • Альтернативные устройства ввода: используются людьми с ограниченными возможностями передвижения для доступа к интернету. К ним относятся такие устройства, как джойстики, трекболы и сенсорные панели.
  • Распознавание голоса: используется людьми с ограниченными возможностями передвижения для доступа к интернету. Оно позволяет пользователям управлять компьютером с помощью голоса.

Инструменты аудита доступности

  • Распространенные инструменты проверки доступности: Google Lighthouse, Wave, IBM Equal Accessibility Checker и axe DevTools — это некоторые из распространенных инструментов, используемых для проверки доступности веб-сайта.

Лучшие практики обеспечения доступности

  • Правильная структура заголовков: Для создания логичной структуры контента следует использовать правильные уровни заголовков. Это помогает людям, использующим вспомогательные технологии, понимать содержимое вашего сайта.
  • Доступность и таблицы: При использовании таблиц следует использовать элемент th для определения ячеек заголовка и элемент td для определения ячеек данных. Это помогает людям, использующим вспомогательные технологии, понять структуру таблицы. С помощью элемента caption можно написать заголовок таблицы, чтобы пользователи, особенно те, кто использует вспомогательные технологии, могли быстро понять назначение и содержание таблицы. Элемент caption следует размещать сразу после открывающего тега элемента table. Таким образом, программы чтения с экрана и другие вспомогательные технологии смогут предоставить дополнительный контекст, озвучивая заголовок перед чтением содержимого.
  • Важность наличия связанных меток для полей ввода: следует использовать элемент label для связывания меток с полями ввода формы. Это помогает людям, использующим вспомогательные технологии, понимать назначение поля ввода.
  • Важность хорошего альтернативного текста: Атрибут alt следует использовать для предоставления текстовой альтернативы изображениям. Это помогает людям, использующим вспомогательные технологии, понимать содержимое изображения.
  • Важность качественного текста ссылки: следует использовать описательный текст ссылки, чтобы помочь пользователям понять ее назначение. Это также помогает людям, использующим вспомогательные технологии, понять цель ссылки.
  • Рекомендации по обеспечению доступности аудио и видео контента: Для людей с нарушениями слуха следует предоставлять субтитры и расшифровки аудио и видео контента. Для людей с нарушениями зрения следует также предоставлять аудиоописания видеоконтента.
  • Атрибут tabindex: используется для того, чтобы сделать элементы фокусируемыми и определить относительный порядок навигации по ним с помощью клавиатуры. Важно никогда не использовать атрибут tabindex со значением больше 0. Вместо этого следует использовать значение 0 или -1. 
    •         
          <p tabindex="-1">Извините, произошла ошибка при отправке вашего сообщения.</p>
  • Атрибут accesskey: используется для определения сочетания клавиш для элемента. Это может помочь пользователям с ограниченными возможностями передвижения легче перемещаться по веб-сайту. 
    •         
          <button accesskey="s">Сохранить</button>
          <button accesskey="c">Отмена</button>
          <a href="index.html" accesskey="h">Главная</a>

WAI-ARIA, роли и атрибуты

  • WAI-ARIA: это аббревиатура от Web Accessibility Initiative - Accessible Rich Internet Applications (Инициатива по обеспечению доступности веб-сайтов — доступные многофункциональные интернет-приложения). Это набор атрибутов, которые можно добавлять к элементам HTML для повышения доступности. Он предоставляет вспомогательным технологиям дополнительную информацию о назначении и структуре контента.
  • Роли ARIA: набор предопределенных ролей, которые можно добавить к элементам HTML для определения их назначения. Это помогает людям, использующим вспомогательные технологии, понимать содержимое веб-сайта. Примеры включают role="tab", role="menu" и role="alert".

Существует шесть основных категорий ролей в ARIA:

  • Роли в структуре документа(Document structure roles): Эти роли определяют общую структуру веб-страницы. Благодаря этим ролям вспомогательные технологии могут понимать взаимосвязи между различными разделами и помогать пользователям ориентироваться в контенте.
  • Роли виджетов(Widget roles): Эти роли определяют назначение и функциональность интерактивных элементов, таких как полосы прокрутки.
  • Роли-ориентиры(Landmark roles): Эти роли классифицируют и обозначают основные разделы веб-страницы. Программы чтения с экрана используют их для удобной навигации по важным разделам страницы.
  • Роли динамически изменяющихся областей(Live region roles): Эти роли определяют элементы с контентом, который будет динамически изменяться. Таким образом, программы чтения с экрана и другие вспомогательные технологии могут сообщать об изменениях пользователям с нарушениями зрения.
  • Роли окон(Window roles): Эти роли определяют дочерние окна, такие как всплывающие модальные диалоги. К таким ролям относятся alertdialog и dialog.
  • Абстрактные роли(Abstract roles): Эти роли помогают организовать документ. Они предназначены только для внутреннего использования браузером, а не разработчиками, поэтому вы должны знать об их существовании, но не должны использовать их на своих веб-сайтах или веб-приложениях.

Атрибуты

  • Атрибуты aria-label и aria-labelledby: Эти атрибуты используются для присвоения элементу программного (или доступного) имени, что помогает людям, использующим вспомогательные технологии (например, программы чтения с экрана), понять назначение элемента. Они часто используются, когда визуальная метка элемента представляет собой изображение или символ, а не текст. Атрибут aria-label позволяет определить имя непосредственно в атрибуте, а aria-labelledby позволяет ссылаться на существующий текст на странице. 
    •       
        <button aria-label="Поиск">
            <i class="fas fa-search"></i>
        </button>
      
    
          
        <input type="text" aria-labelledby="search-btn">
        <button type="button" id="search-btn">Поиск</button>
  • Атрибут aria-hidden: используется для скрытия элемента от вспомогательных технологий, таких как программы чтения с экрана. Например, его можно использовать для скрытия декоративных изображений, не содержащих никакого осмысленного контента. 
    •       
        <button>
            <i class="fa-solid fa-gear" aria-hidden="true"></i>
            <span class="label">Настройки</span>
        </button>
  • Атрибут aria-describedby: Используется для предоставления дополнительной информации об элементе путем связывания его с другим элементом, содержащим эту информацию. Это дает пользователям программ чтения с экрана немедленный доступ к дополнительной информации при переходе к элементу. Часто используется, например, для связывания инструкций по форматированию с текстовым полем ввода или сообщения об ошибке с полем ввода после некорректной отправки формы. 
    •       
        <form>
            <label for="password">Пароль:</label>
            <input type="password" id="password" aria-describedby="password-help" />
            <p id="password-help">Ваш пароль должен состоять как минимум из 8 символов.</p>
        </form>
