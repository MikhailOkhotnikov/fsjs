Базовый обзор HTML

Основы HTML

  • Роль HTML: HTML представляет содержание и структуру веб-страницы.
  • HTML-элементы: Элементы — это строительные блоки HTML-документа. Они представляют собой заголовки, абзацы, ссылки, изображения и многое другое. Большинство HTML-элементов состоят из открывающего тега (<elementName>) и закрывающего тега (</elementName>).

    • Вот основной синтаксис:

          
        <elementName>Содержимое будет здесь</elementName>
  • Пустые элементы: пустые элементы не могут содержать никакого контента и имеют только начальный тег. Примерами служат элементы img и meta.
    •       
        <img>
        <meta>

    В некоторых кодовых базах часто встречается косая черта / внутри пустого элемента. Оба варианта допустимы: 

          
        <img>
        <img/>
  • Атрибуты: Атрибут — это значение, помещённое в открывающий тег HTML-элемента. Атрибуты предоставляют дополнительную информацию об элементе или определяют его поведение. Ниже представлен базовый синтаксис атрибута:
    •       
        <element attribute="value"></element>

    Булев атрибут — это атрибут, который может присутствовать или отсутствовать в HTML-теге. Если присутствует, значение — true, в противном случае — false. Примеры булевых атрибутов: disabled, readonly и required.

  • Комментарии: Комментарии используются в программировании для того, чтобы оставлять заметки для себя и других разработчиков в коде. Вот синтаксис комментария в HTML:
    •       
        <!--Это HTML-комментарий.-->

Общие элементы HTML

  • Элементы заголовков: в HTML существует шесть элементов заголовков. Заголовки от h1 до h6 используются для обозначения важности контента, находящегося под ними. Чем меньше число, тем выше важность, поэтому элементы h2 менее важны, чем элементы h1.
    •       
        <h1>самый важный элемент заголовка</h1>
        <h2>второй по важности элемент заголовка</h2>
        <h3>третий по важности элемент заголовка</h3>
        <h4>четвертый по важности элемент заголовка</h4>
        <h5>пятый по важности элемент заголовка</h5>
        <h6>наименее важный элемент заголовка</h6>
  • Элементы абзаца: используются для абзацев на веб-странице.
    •       
        <p>Это элемент абзаца.</p>
  • Элементы img: Элемент img используется для добавления изображений на веб-страницу. Атрибут src указывает местоположение изображения. Для элементов изображений рекомендуется включать ещё один атрибут — атрибут alt. Вот пример элемента img с атрибутами src и alt:
    •       
        <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="Кусочек лазаньи на тарелке.">
  • Элемент body: этот элемент используется для представления содержимого HTML-документа.
    •       
        <body>
          <h1>Приложение CatPhoto</h1>
          <p>Это элемент абзаца.</p>
        </body>
  • Элементы section: этот элемент используется для разделения содержимого на более мелкие разделы.
    •       
        <section>
          <h2>Обо мне</h2>
          <p>Привет, меня зовут Джейн Доу, и я веб-разработчик.</p>
        </section>
  • Элементы div: Этот элемент — стандартный HTML-элемент, не несущий никакого семантического значения. Он используется как стандартный контейнер для других HTML-элементов.
    •       
        <div>
          <h1>Я - заголовок</h1>
          <p>Я - абзац</p>
        </div>
  • Элементы привязки (a): Эти элементы используются для размещения ссылок на веб-странице. Атрибут href используется для указания того, куда должна переходить ссылка, когда пользователь нажимает на нее.
    •       
        <a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">милые котики</a>
  • Элементы неупорядоченного (ul) и упорядоченного (ol) списка: Чтобы создать маркированный список элементов, следует использовать элемент ul с одним или несколькими элементами li, вложенными внутрь, следующим образом:
    •       
        <ul>
          <li>кошачья мята</li>
          <li>лазерные указки</li>
          <li>лазанья</li>
        </ul>

    Для создания упорядоченного списка элементов следует использовать элемент ol:

          
        <ol>
          <li>средство от блох</li>
          <li>гром</li>
          <li>другие кошки</li>
        </ol>
  • Элемент выделения (em): используется для выделения(emphasis) фрагмента текста.
    •       
        <p>Кошки <em>любят</em> лазанью.</p>
  • Элемент «Сильная важность» (strong): этот элемент используется для придания тексту особого значения, чтобы подчеркнуть его срочность и серьезность.
    •       
        <p>
          <strong>Важно:</strong> Прежде чем продолжить, обязательно наденьте защитные очки. 
        </p>
  • Элементы figure и figcaption: элемент figure используется для группировки контента, например изображений и диаграмм. Элемент figcaption используется для представления подписи к этому контенту внутри элемента
    •       
        <figure>
          <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Два полосатых котенка спят вместе на диване.">
          <figcaption>Кошки <strong>ненавидят</strong> других кошек.</figcaption>  
        </figure>
  • Элемент main: этот элемент используется для представления основного(main) содержимого веб-страницы.
    •       
        <body>
          <main>
            Основное содержимое веб-страницы.
          </main>
        </body>
  • Элемент footer: этот элемент размещается в нижней части HTML-документа и обычно содержит информацию об авторских правах и другие важные ссылки на страницы.
    •       
        <footer>
          <p>
            No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
          </p>
        </footer>

Идентификаторы и группировка

  • Идентификаторы(id): уникальные идентификаторы элементов HTML. Имена идентификаторов следует использовать только один раз в HTML-документе.
    •       
        <h1 id="title">Страница обзора фильма</h1>

    В названии идентификатора не должно быть пробелов. Если ваш идентификатор состоит из нескольких слов, вы можете использовать дефисы между ними, например: 

          
        <div id="red-box"></div>
  • Классы(class): Классы используются для группировки элементов по стилю и поведению.
    •       
        <div class="box"></div>

    В отличие от идентификаторов, одно и то же имя класса можно использовать повторно во всем HTML-документе. Значение класса также может содержать пробелы, например: 

          
        <div class="box red-box"></div>
        <div class="box blue-box"></div>

Специальные символы и ссылки

  • HTML-сущности(entities): HTML-сущность, или ссылка на символ, — это набор символов, используемых для представления зарезервированного символа в HTML. Примеры включают символ амперсанд(&amp;) и символ меньше(&lt;) или больше(&gt;)
    •       
        <p>Это элемент &lt;img /&gt; </p>
  • Элемент ссылки(link): Этот элемент используется для ссылки на внешние ресурсы, такие как таблицы стилей и значки сайтов. Ниже приведен базовый синтаксис использования элемента link для внешнего CSS-файла:
    •       
        <link rel="stylesheet" href="./styles.css" />

    Атрибут rel используется для определения связи между связанным ресурсом и HTML-документом. Атрибут href используется для указания расположения URL-адреса внешнего ресурса.

  • Элемент script: этот элемент используется для внедрения исполняемого кода.
    •       
        <body>
          <script>
            alert("Добро пожаловать на freeCodeCamp");
          </script>
        </body>

    Хотя технически вы можете разместить весь код JavaScript внутри тегов script, рекомендуется вместо этого ссылаться на внешний файл JavaScript. Вот пример использования элемента script для ссылки на внешний файл JavaScript: 

          
        <script src="path-to-javascript-file.js"></script>

    Атрибут src используется здесь для указания местоположения внешнего файла JavaScript.

Шаблон и кодирование

  • Шаблон(boilerplate) HTML: это шаблон, включающий базовую структуру и основные элементы, необходимые каждому HTML-документу.
    •       
        <!DOCTYPE html>
        <html lang="ru">
          <head>
            <meta charset="utf-8" />
            <title>freeCodeCamp</title>
            <link rel="stylesheet" href="./styles.css" />
          </head>
          <body>
            <!--Заголовки, абзацы, изображения и т.д. можно найти здесь-->
          </body>
        </html>
  • Элемент DOCTYPE: Этот тег используется для указания браузерам, какую версию HTML вы используете.
  • Элемент html: представляет элемент верхнего уровня или корень HTML-документа. Чтобы указать язык документа, используйте атрибут lang.
  • Элемент head: раздел head содержит важные метаданные, то есть скрытую информацию, необходимую браузерам и поисковым системам.
  • Элементы meta: эти элементы представляют метаданные вашего сайта. Они содержат информацию о таких вещах, как кодировка символов, а также о том, как сайты, такие как Вконтакте, должны отображать ссылку на вашу страницу и многое другое.
  • Элемент title: этот элемент используется для установки текста, который отображается в качестве заголовка во вкладке или окне браузера.
  • Кодировка символов UTF-8: UTF-8, или UCS Transformation Format 8, — это стандартизированная кодировка символов, широко используемая в Интернете. Кодировка символов — это метод, используемый компьютерами для хранения символов в виде данных. Атрибут charset используется внутри элемента meta для установки кодировки символов UTF-8.

SEO и социальные сети

  • SEO: поисковая оптимизация — это практика оптимизации веб-страниц, в результате которой они становятся более заметными и занимают более высокие позиции в результатах поиска.
  • Meta элемент description: используется для предоставления краткого описания веб-страницы и влияет на SEO.
    •       
        <meta
          name="description"
          content="Узнайте о советах экспертов и методах озеленения небольших помещений, правильного выбора растений и поддержания цветущего сада."
        />
  • Теги Open Graph: протокол Open Graph позволяет вам контролировать, как контент вашего веб-сайта отображается на различных платформах социальных сетей, таких как Вконтакте, LinkedIn и многих других.

    • Задавая эти свойства Open Graph, вы можете побудить пользователей кликнуть по ссылке и взаимодействовать с вашим контентом. Эти свойства можно задать с помощью набора элементов meta в HTML-разделе head.

  • Свойство og:title: используется для установки заголовка, отображаемого для публикаций в социальных сетях.
    •       
        <meta content="freeCodeCamp.org" property="og:title" />
  • Свойство og:type: Свойство type используется для представления типа контента, публикуемого в социальных сетях. Примерами такого контента являются статьи, веб-сайты, видео или музыка.
    •       
        <meta property="og:type" content="website" />
  • Свойство og:image: используется для установки изображения, отображаемого для публикаций в социальных сетях.
    •       
        <meta
          content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
          property="og:image"
        />
  • Свойство og:url: используется для установки URL-адреса, по которому пользователи будут переходить к публикациям в социальных сетях.
    •       
        <meta property="og:url" content="https://www.freecodecamp.org" />

Медиа-элементы и оптимизация

  • Заменяемые элементы: Заменяемый элемент — это элемент, содержимое которого определяется внешним ресурсом, а не самим CSS. Примером может служить элемент iframe. iframe означает встроенный фрейм (inline frame). Это встроенный элемент, используемый для встраивания другого HTML-контента непосредственно в HTML-страницу.
    •       
        <iframe src="https://www.example.com" title="Пример сайта"></iframe>

    Вы можете включить атрибут allowfullscreen, который позволяет пользователю отображать iframe в полноэкранном режиме. 

          
        <iframe
          src="video-url"
          width="width-value"
          height="height-value"
          allowfullscreen
        ></iframe>

    Чтобы встроить видео в iframe, вы можете скопировать его напрямую с популярных видеосервисов, таких как YouTube и Vimeo, или определить его самостоятельно с помощью атрибута src, указывающего на URL-адрес этого видео. Вот пример встраивания популярного курса freeCodeCamp с YouTube: 

          
        <h1>Видеоролик freeCodeCamp на YouTube, встроенный в элемент iframe</h1>

        <iframe
          width="560"
          height="315"
          src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
          title="YouTube video player"
          allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
          referrerpolicy="strict-origin-when-cross-origin"
          allowfullscreen
        ></iframe>

    Существуют и другие заменяемые элементы, такие как video и embed. Некоторые элементы ведут себя как заменяемые элементы при определённых обстоятельствах. Вот пример элемента input с атрибутом type, установленным на image: 

          
        <input type="image" alt="Описательный текст находится здесь" src="example-img-url">
  • Оптимизация медиаконтента: При использовании медиаконтента, например изображений, на веб-страницах следует учитывать три фактора: размер, формат и сжатие. Алгоритм сжатия используется для уменьшения размера файлов или данных.
  • Форматы изображений: Два наиболее распространённых формата файлов — PNG и JPG, но они уже не самые идеальные форматы для показа изображений. Если вам не нужна поддержка старых браузеров, рассмотрите возможность использования более оптимизированного формата, например, WEBP или AVIF.
  • Лицензии изображений: Изображение, находящееся в общественном достоянии, не защищено авторскими правами и может использоваться без каких-либо ограничений. Изображения, лицензированные по лицензии Creative Commons 0, считаются общественным достоянием. Некоторые изображения могут быть выпущены по разрешительной лицензии, например, по лицензии Creative Commons или лицензии BSD, которую использует freeCodeCamp.
  • SVG: масштабируемая векторная графика(Scalable Vector Graphics) отслеживает данные на основе траекторий и уравнений для построения точек, линий и кривых. Фактически это означает, что векторную графику, такую ​​как SVG, можно масштабировать до любого размера без потери качества.

Интеграция мультимедиа

  • Элементы audio и video: Элементы аудио и видео позволяют добавлять аудио и видео контент в HTML-документы. Элемент audio поддерживает популярные аудиоформаты, такие как MP3, WAV и OGG. Элемент video поддерживает форматы MP4, OGG и WebM.
    •       
        <audio src="CrystalizeThatInnerChild.mp3"></audio>

    Если вы хотите видеть аудиоплеер на странице, то вы можете добавить элемент audio с атрибутом controls: 

          
        <audio src="CrystalizeThatInnerChild.mp3" controls></audio>

    Атрибут controls позволяет пользователям управлять воспроизведением звука, включая регулировку громкости, приостановку и возобновление воспроизведения. Атрибут controls — это логический атрибут, который можно добавить к элементу для включения встроенных элементов управления воспроизведением. Если он не указан, элементы управления отображаться не будут.

  • Атрибут цикла(loop): Атрибут loop — это логический атрибут, который заставляет звук воспроизводиться непрерывно.
    •       
        <audio
         src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
         loop
         controls
        ></audio>
  • Атрибут muted: При наличии в элементе audio логического атрибута muted(приглушенный) воспроизведение звука будет отключено.
    •       
        <audio
         src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
         loop
         controls
         muted
        ></audio>
  • Элемент source: Что касается типов аудиофайлов, браузеры поддерживают разные типы по-разному. Чтобы это учесть, можно использовать элементы source внутри элемента audio, и браузер выберет первый распознанный им источник. Вот пример использования нескольких элементов source для элемента audio:
    •       
        <audio controls>
          <source src="audio.ogg" type="audio/ogg" />
          <source src="audio.wav" type="audio/wav" />
          <source src="audio.mp3" type="audio/mpeg" />
        </audio>

    Все атрибуты, которые мы изучили до сих пор, также поддерживаются элементом video. Вот пример использования элемента video с атрибутами loop, controls и muted: 

          
        <video
          src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
          loop
          controls
          muted
        ></video>
  • Атрибут poster: Если вы хотите отображать изображение во время загрузки видео, вы можете использовать атрибут poster. Этот атрибут недоступен для элементов audio и уникален для элемента video.
    •       
        <video
          src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
          loop
          controls
          muted
          poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
          width="620"
        ></video>

Типы атрибутов target

  • Атрибут target: Этот атрибут сообщает браузеру, где открыть URL-адрес элемента привязки. Существует четыре важных возможных значения этого атрибута: _self, _blank, _parent и _top. Существует пятое значение, _unfencedTop, которое в настоящее время используется для экспериментального API FencedFrame. Вероятно, у вас пока нет причин использовать его.
  • Значение _self: это значение по умолчанию для атрибута target. Ссылка открывается в текущем контексте браузера. В большинстве случаев это текущая вкладка или окно.
    •       
        <a href="https://freecodecamp.org" target="_self">Посетите freeCodeCamp</a>
  • Значение _blank: открывает ссылку в новом контексте браузера. Обычно это происходит в новой вкладке. Но некоторые пользователи могут настроить свои браузеры на открытие нового окна.
    •       
        <a href="https://freecodecamp.org" target="_blank">Посетите freeCodeCamp</a>
  • Значение _parent: открывает ссылку в родительском элементе текущего контекста. Например, если на вашем сайте есть iframe, значение _parent в этом iframe откроется во вкладке/окне вашего сайта, а не во встроенном фрейме.
    •       
        <a href="https://freecodecamp.org" target="_parent">Посетите freeCodeCamp</a>
  • Значение _top: открывает ссылку в самом верхнем контексте браузера — «родительском элементе родительского элемента». Аналогично _parent, но ссылка всегда будет открываться в полной вкладке/окне браузера, даже для вложенных встроенных фреймов.
    •       
        <a href="https://freecodecamp.org" target="_top">Посетите freeCodeCamp</a>

Абсолютные и относительные пути

  • Определение пути: Путь(path) — это строка, указывающая расположение файла или каталога в файловой системе. В веб-разработке пути позволяют разработчикам ссылаться на ресурсы, такие как изображения, таблицы стилей, скрипты и другие веб-страницы.
  • Синтаксис пути: необходимо знать три основных синтаксиса. Первый — это косая черта, которая может быть обратной (\) или прямой (/) в зависимости от операционной системы. Второй — одинарная точка (.). И наконец, двойная точка (..). Косая черта известна как «разделитель пути». Она используется для обозначения разрыва текста между именами папок или файлов. Одна точка указывает на текущий каталог, а две — на родительский.
    •       
        public/index.html
        ./favicon.ico
        ../src/index.css
  • Абсолютный путь: Абсолютный путь(absolute path) — это полная ссылка на ресурс. Он начинается с корневого каталога, включает все остальные каталоги и, наконец, имя файла и расширение. «Корневой каталог» относится к каталогу или папке верхнего уровня в иерархии. Абсолютный путь также включает протокол (это может быть http, https, file) и доменное имя, если ресурс находится в сети. Вот пример абсолютного пути, ведущего к логотипу freeCodeCamp:
    •       
        <a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
          Просмотр логотипа fCC
        </a>
  • Относительный путь: Относительный путь(relative path) указывает расположение файла относительно каталога текущего файла. Он не включает протокол и доменное имя, что делает его более коротким и гибким для внутренних ссылок в пределах одного веб-сайта. Вот пример ссылки на страницу about.html со страницы contact.html, которые находятся в одной папке:
    •       
        <p>
          Читайте больше 
          <a href="about.html">о нас</a>
        </p>

Состояния ссылки

  • :link: Это состояние по умолчанию. Оно представляет ссылку, по которой пользователь ещё не перешёл, не нажал и не взаимодействовал с ней. Это состояние можно рассматривать как базовый набор стилей для всех ссылок на странице. Остальные состояния строятся на его основе.
  • :visited: Это применяется, когда пользователь уже посетил страницу, на которую ссылается ссылка. По умолчанию ссылка становится фиолетовой, но вы можете использовать CSS, чтобы предоставить пользователю другое визуальное обозначение.
  • :hover: Это состояние применяется, когда пользователь наводит курсор на ссылку. Оно полезно для привлечения дополнительного внимания к ссылке, чтобы убедиться, что пользователь действительно намерен нажать на неё.
  • :focus: Это состояние применяется, когда мы фокусируемся на ссылке.
  • :active: Это состояние применяется к ссылкам, активируемым пользователем. В большинстве случаев это означает нажатие на ссылку левой кнопкой мыши.
