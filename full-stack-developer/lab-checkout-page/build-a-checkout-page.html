Создать страницу оформления заказа

Создайте приложение, функционально похожее на этот пример проекта. Старайтесь не копировать пример проекта, придайте ему свой собственный стиль.

Цель: Выполнить приведенные ниже пользовательские истории и добиться успешного прохождения всех тестов для завершения лабораторной работы.

Истории пользователей:

  1. Вам нужен элемент h1 с текстом Оформить заказ.
  2. Сразу после элемента h1 должны располагаться два элемента section.
  3. В первом разделе должен быть элемент h2 с текстом Ваша корзина.
  4. В первом разделе должно быть изображение элемента с соответствующим альтернативным текстом. При желании вы можете использовать это изображение: https://cdn.freecodecamp.org/curriculum/labs/cube.jpg
  5. Во втором разделе должен быть элемент h2 с текстом Информация об оплате.
  6. Во втором разделе должен быть элемент form.
  7. В вашей форме должен быть элемент ввода с id и name равными card-name, а также type равным text и связанная с ним метка(label).
  8. В вашей форме должен быть элемент ввода с id и name равными card-number, а также type равным text и связанная с ним метка(label).
  9. У вас должно быть как минимум два элемента input с атрибутом required.
  10. Для каждого обязательного поля ввода следует разместить внутри элемента label элемент span с текстом * и атрибутом aria-hidden, установленным в true, чтобы обязательное поле было визуально обозначено.
  11. После поля ввода номера карты должен быть элемент p с текстом справки, поясняющим необходимый формат номера карты. Этот элемент должен иметь идентификатор card-number-help и ссылаться на поле ввода номера карты с помощью атрибута aria-describedby.

Тесты

  1. Вам нужен элемент h1 с текстом Оформить заказ.
  2. На вашей странице должен быть только один элемент h1.
  3. После элемента h1 должно быть как минимум два элемента section.
  4. В первом разделе должен быть элемент h2 с текстом Ваша корзина.
  5. В первом разделе должно быть как минимум одно изображение с альтернативным текстом.
  6. Во втором разделе должен быть элемент h2 с текстом Информация об оплате.
  7. Внутри второго элемента section должен находиться элемент form.
  8. В вашей форме должен быть элемент input с id и name равными card-name.
  9. В поле ввода названия карточки должен быть указан type равный text.
  10. В вашей форме должен быть элемент input с id и name равными card-number.
  11. В поле ввода номера карты должен быть указан type равный text.
  12. Все элементы input, кроме элементов c type равным submit, должны быть связаны с элементом label.
  13. У вас должно быть как минимум два элемента input с атрибутом required.
  14. Для каждого обязательного поля ввода следует добавить элемент span с текстом * внутри элемента label.
  15. Для элементов span следует установить свойство aria-hidden в значение true.
  16. Сразу после поля ввода номера карты должен располагаться элемент p с id равным card-number-help.
  17. Текст справки по номеру карты не должен быть пустым.
  18. Для поля ввода номера карты необходимо установить атрибут aria-describedby в значение card-number-help.
