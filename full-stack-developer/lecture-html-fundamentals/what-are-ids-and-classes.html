Что такое идентификаторы и классы и когда следует их использовать?

Атрибут id добавляет уникальный идентификатор HTML-элементу. В этом примере есть элемент h1 с идентификатором title: 

    
      <h1 id="title">Страница с обзором фильма</h1>

Вы можете ссылаться на идентификатор(id) заголовка(title) в JavaScript или CSS. Вот пример CSS, где идентификатор заголовка используется для изменения цвета(color) текста на красный(red): 

    
      #title {
        color: red;
      }

Хэш-символ (#) перед title сообщает компьютеру, что вы хотите настроить таргетинг на идентификатор с этим значением. Значения идентификаторов(id) не следует использовать более одного раза, и они всегда должны быть уникальными. Еще одна вещь, на которую следует обратить внимание в отношении значений идентификаторов, заключается в том, что в них не должно быть пробелов. Вот пример применения слов main и heading к значению атрибута id: 

    
      <h1 id="main heading">Основной заголовок</h1>

Браузеры будут воспринимать этот пробел как часть идентификатора, что приведет к нежелательным проблемам при стилизации и написании скриптов. Значения атрибута id должны содержать только буквы, цифры, подчеркивания и тире.


В отличие от атрибута id, значение атрибута class не обязательно должно быть уникальным и может содержать пробелы. Вот пример применения класса box к элементу div: 

    
      <div class="box"></div>

Если вы хотите добавить к элементу несколько имён классов, разделите их пробелом. Вот обновлённый пример применения нескольких классов к элементу div: 

    
      <div class="box red-box"></div>

Чтобы применить набор стилей к этому классу box, вы можете сослаться на него в CSS-коде. Вот пример настройки свойств ширины(width), высоты(height) и границы(border) для элемента с именем класса box: 

    
      .box {
        width: 200px;
        height: 200px;
        border: 2px solid black;
      }

Символ точки (.) перед box сообщает компьютеру, что вы хотите настроить таргетинг на класс с этим значением. При запуске кода он выполнит поиск по всему HTML-документу, чтобы найти все элементы с этим именем класса и применить эти стили.

Итак, подведем итог: когда следует использовать идентификатор(id), а когда — класс(class)? Классы лучше всего использовать, когда нужно применить набор стилей ко многим элементам. Если вы хотите настроить таргетинг на конкретный элемент, лучше всего использовать идентификатор, поскольку эти значения должны быть уникальными.
