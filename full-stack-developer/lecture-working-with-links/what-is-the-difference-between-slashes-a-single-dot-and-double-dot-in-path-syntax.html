В чем разница между косыми чертами, одинарной точкой и двойной точкой в ​​синтаксисе пути?

Возможно, вы уже встречали ссылки типа /public/logo.png, ./script.js или ../styles.css. Но что означают эти особые типы ссылок? Они называются путями к файлам. Существует три ключевых синтаксиса, которые нужно знать. Первый — это косая черта, которая может быть обратной (\) или прямой (/) в зависимости от вашей операционной системы. Второй — одинарная точка (.). И, наконец, двойная точка (..).

Слэш называется «разделителем пути». Он используется для обозначения разрыва в тексте между именами папок или файлов. Благодаря этому ваш компьютер понимает, что naomis-files/ указывает на каталог с тем же именем, а naomis/files/ — на каталог с файлами(files) в каталоге naomis.

Одна точка указывает на текущий каталог, а две точки — на родительский. Обычно одна точка используется для того, чтобы гарантировать, что путь распознаётся как относительный. Вспомните, что на предыдущей лекции вы изучали разницу между относительными и абсолютными путями.

Однако двойные точки гораздо чаще используются для доступа к файлам за пределами текущего рабочего каталога.

Например, рассмотрим следующее файловое дерево:

    
      my-app/
      ├─ public/
      │  ├─ favicon.ico
      │  ├─ index.html
      ├─ src/
      │  ├─ index.css
      │  ├─ index.js

Если файл public/index.html должен загружать файл favicon.ico, для доступа к текущему каталогу используйте относительный путь с одной точкой: ./favicon.ico. Если вашему файлу public/index.html необходимо загрузить файл index.css, вы должны использовать относительный путь с двумя точками, чтобы сначала перейти к родительскому каталогу my-app, затем к каталогу src и, наконец, к вашему файлу: ../src/index.css.
