Каковы различные типы целевых атрибутов и как они работают?

Возможно, вы видели атрибут target в элементах-якорях(anchor) или ссылках. Этот важный атрибут сообщает браузеру, где открыть URL-адрес элемента-якоря: 

    
      <a href="https://freecodecamp.org" target="_blank">Посетите freeCodeCamp</a>

Для этого атрибута существует четыре важных возможных значения. Обратите внимание, что каждому значению предшествует символ подчеркивания.

Первое значение — _self, что является значением по умолчанию. Ссылка открывается в текущем контексте браузера. В большинстве случаев это текущая вкладка или окно.

Второе значение — _blank, что означает, что ссылка открывается в новом контексте браузера. Обычно это происходит в новой вкладке. Но некоторые пользователи могут настроить свои браузеры на открытие нового окна.

Третье значение — _parent, которое открывает ссылку в родительском элементе текущего контекста. Например, если на вашем сайте есть iframe, значение _parent в этом iframe откроется во вкладке/окне вашего сайта, а не во встроенном фрейме.

Четвёртое значение — _top, которое открывает ссылку в самом верхнем контексте браузера — «родитель родителя». Это похоже на _parent, но ссылка всегда будет открываться в полной вкладке/окне браузера, даже для вложенных фреймов.

Пятое значение, _unfencedTop, в настоящее время используется для экспериментального API FencedFrame. На момент прочтения этой лекции у вас, вероятно, ещё не будет причин использовать его.

Выбор правильного значения target для контроля конечного пункта назначения пользователей является важным фактором при создании веб-сайта.
