Как можно сделать веб-приложения доступными с помощью клавиатуры?

Многие пользователи предпочитают клавиатуру мыши из-за физических ограничений, травм от повторяющихся движений или личных предпочтений. Это касается как пользователей программ чтения с экрана, так и тех, у кого нет работающей мыши. Доступность клавиатуры гарантирует, что эти пользователи смогут эффективно перемещаться по веб-приложениям без каких-либо препятствий.

Рассмотрим несколько практических методов, которые можно использовать для обеспечения доступности веб-приложений с помощью клавиатуры.

Многие пользователи используют клавишу Tab для перемещения по интерактивным элементам на веб-странице. По умолчанию браузеры позволяют пользователям перемещаться по элементам, таким как ссылки, кнопки и поля форм, в том порядке, в котором они отображаются в HTML-коде. Это называется естественным порядком перехода по клавише Tab.

Иногда может потребоваться настроить, какие элементы будут доступны для фокусировки, или изменить порядок их фокусировки. Атрибут tabindex позволяет это сделать.

Вот базовый синтаксис: 

    
      <element tabindex="число">Текст элемента</element>

Значение параметра tabindex определяет поведение элемента при навигации с помощью клавиатуры:

tabindex="0" добавляет элемент к естественному порядку перехода по клавишам.

Переключение с помощью клавиши Tab позволит перемещать фокус с button на элемент div, а затем на ссылку, в порядке их расположения в HTML-коде.

Щелкните в любом месте пустого пространства окна предварительного просмотра. Затем используйте клавишу Tab, чтобы увидеть, как фокус перемещается между различными элементами. Для взаимодействия с примером вам потребуется включить интерактивный редактор. 

    
      <button>Первый</button>
      <div tabindex="0">Второй</div>
      <a href="#">Третий</a>

tabindex="-1" позволяет программно установить фокус на элемент. Это полезно для управления фокусом в элементах, которые обычно не могут быть сфокусированы, таких как заголовки, контейнеры, диалоговые окна или сообщения об ошибках: 

    
      <p tabindex="-1">Извините, произошла ошибка при отправке вашего сообщения.</p>

В этом примере абзац не входит в обычный порядок перехода по клавише Tab, поэтому пользователи не могут получить к нему доступ, нажав клавишу Tab. Однако, если вы установите фокус на этот элемент с помощью скрипта, сообщение будет доведено до сведения пользователя. Подробнее об этом методе вы узнаете в уроках по JavaScript.

Когда значение tabindex больше 0, устанавливается пользовательский порядок перехода по вкладкам. Таким образом, элементы с меньшими положительными значениями получают фокус первыми.

В этом примере нажатие клавиши Tab сначала сфокусируется на input с индексом tabindex="1", затем на поле с индексом tabindex="2", а затем на поле с индексом tabindex="3", независимо от их порядка в HTML. 

    
      <input tabindex="2">
      <input tabindex="1">
      <input tabindex="3">

В сложных виджетах, например, в панели инструментов, где требуется определенный порядок навигации, иногда используются пользовательские положительные значения. Однако такой подход не рекомендуется, поскольку он может сделать навигацию запутанной и сложной в обслуживании, особенно по мере роста или изменения страницы.

accesskey — ещё один атрибут, который можно использовать для обеспечения доступности веб-проекта с помощью клавиатуры. Он позволяет определить клавишу, которая фокусируется на определённом элементе или активирует его.

Вот интерактивный пример, который вы можете попробовать, следуя приведенным ниже рекомендациям (предварительно включите интерактивный редактор):

  • accesskey="s" назначает клавишу S кнопке Сохранить. В большинстве браузеров эта кнопка активируется нажатием ALT + S (в Windows) и CTRL + Option + S (в Mac).
  • accesskey="c" назначает клавишу C на кнопку Отмена, позволяя пользователям активировать её с помощью ALT + C (Windows) и CTRL + Option + C (Mac).
  • accesskey="h" назначает клавишу H для ссылки Главная(Home ), позволяя пользователям переходить на главную страницу с помощью ALT + H (Windows) и CTRL + Option + H (Mac).

Обратите внимание, что точная комбинация клавиш для активации клавиши доступа может различаться в зависимости от браузера и операционной системы. Обычно это ALT + указанная клавиша в Windows и CTRL + Option + указанная клавиша на Mac. 

    
      <button accesskey="s">Сохранить</button>
      <button accesskey="c">Отмена</button>
      <a href="index.html" accesskey="h">Главная</a>

Ещё один способ сделать клавиатуру доступной в ваших приложениях — обеспечить чёткие индикаторы фокуса. Если вы считаете, что стандартного индикатора фокуса браузера недостаточно, вы можете переопределить его, обратившись к состоянию фокуса элемента.

Вот пример изменения состояния фокуса для кнопки. Щелкните в любом месте пустого пространства окна предварительного просмотра, а затем нажмите клавишу Tab, чтобы сфокусировать button. Для взаимодействия с примером необходимо включить интерактивный редактор.

Файл index.html 
      
        <link href="styles.css" rel="stylesheet">
        <button>Пример кнопки</button>
Файл styles.css 
      
        button:focus {
          outline: 2px solid #005fcc;
        }

Свойство outline используется для определения контура вокруг элемента. В этом примере контур задан в виде сплошной синей линии толщиной 2 пикселя. Индикатор фокуса должен быть оформлен таким образом, чтобы было очевидно, какой элемент в данный момент находится в фокусе. Для обеспечения доступности индикатор должен иметь минимальный цветовой контраст не менее 3:1 с цветом фона, который он покрывает.

Также следует избегать «ловушек клавиатуры», которые возникают, когда пользователь не может переключить фокус с определенного элемента в таких компонентах, как модальные окна и всплывающие окна.
