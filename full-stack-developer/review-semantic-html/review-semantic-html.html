Обзор cемантического HTML

Важность семантического HTML

  • Структурная иерархия элементов заголовков: Для поддержания структурной иерархии контента важно использовать правильный элемент заголовка. Элемент h1 — это заголовок самого высокого уровня, а элемент h6 — самого низкого.
  • Презентационные элементы HTML: элементы, определяющие внешний вид содержимого. Например, устаревшие элементы center, big и font.
  • Семантические элементы HTML: элементы, несущие смысл и структуру. Например, header, nav, figure.

Семантические элементы HTML

  • Элемент заголовка(header): используется для определения заголовка документа или раздела.
  • Основной элемент(main): используется для размещения основного содержимого веб-страницы.
  • Элемент раздела(section): используется для разделения содержимого на более мелкие разделы.
  • Элемент раздел навигации(nav): представляет собой раздел с навигационными ссылками.
  • Элемент рисунка(figure): используется для размещения иллюстраций и диаграмм.
  • Элемент выделения (em): выделяет текст, на котором есть акцент. 
            
          <p>
            Никогда не отказывайся от <em>своей</em> мечты.
          </p>
  • Элемент идиоматического текста (i): используется для выделения альтернативного голоса или настроения, идиоматических терминов из другого языка, технических терминов и мыслей. 
            
          <p>
            В воздухе витает некое <i lang="fr">je ne sais quoi</i>.
          </p>

    Атрибут lang внутри открытого тега i используется для указания языка контента. В данном случае это будет французский. Элемент i не указывает, важен ли текст, а лишь указывает на его отличие от окружающего текста.

  • Элемент Сильная важность(strong): отмечает текст, имеющий большую важность. 
            
          <p>
            <strong>Предупреждение:</strong> Этот продукт может вызывать аллергические реакции.
          </p>
  • Элемент Привлечь внимание(b): используется для привлечения внимания к тексту, не имеющему значения для смысла контента. Обычно используется для выделения ключевых слов в аннотациях или названий продуктов в обзорах. 
            
          <p>
            Мы протестировали несколько продуктов, включая 
            <b>SuperSound 3000</b> для проверки качества звука, 
            <b>QuickCharge Pro</b> для быстрой зарядки и 
            <b>Ecoclean Vacuum</b> для уборки. 
            Первые два показали себя хорошо, но <b>Ecoclean Vacuum</b> не оправдал ожиданий.
          </p>
  • Элемент списка описаний(dl): используется для представления списка групп описаний терминов.
  • Элемент описания термина(dt): используется для представления определяемого термина.
  • Элемент подробностей описания(dd): используется для представления описания термина. 
    •       
        <dl>
          <dt>HTML</dt>
          <dd>Язык гипертекстовой разметки</dd>
          <dt>CSS</dt>
          <dd>Каскадные таблицы стилей</dd>
        </dl>
  • Элемент Блочная цитата(blockquote): используется для обозначения фрагмента, цитируемого из другого источника. Этот элемент имеет атрибут cite. Значение атрибута cite — URL-адрес источника. 
            
          <blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
            "Можете ли вы представить, каково это - быть успешным разработчиком? Создавать программные системы, на которые люди полагаются?"
          </blockquote>
  • Элемент цитата(cite): используется для визуального указания источника цитируемой работы. Размечает название источника. 
            
          <div>
            <blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
              "Можете ли вы представить, каково это - быть успешным разработчиком? Создавать программные системы, на которые люди полагаются?"
            </blockquote>
            <p>
              -Quincy Larson, <cite>Как научиться программировать и получить работу разработчика [Полная книга].</cite>
            </p>
          </div>
  • Элемент встроенной цитаты(q): используется для представления короткой встроенной цитаты. 
            
          <p>
            Как сказал Quincy Larson,
            <q cite="https://www.freecodecamp.org/news/learn-to-code-book/">
              Импульс - это все.
            </q>
          </p>
  • Элемент Аббревиатура(abbr): используется для представления аббревиатуры или акронима. Чтобы помочь пользователям понять, что представляет собой аббревиатура или акроним, можно указать её полную форму, то есть понятное для человека описание, с помощью атрибута title. 
            
          <p>
            <abbr title="HyperText Markup Language">HTML</abbr> — это основа Интернета.
          </p>
  • Элемент Контактный адрес(address): используется для представления контактной информации.
  • Элемент Дата и Время (time): используется для представления даты и/или времени. Атрибут datetime используется для перевода даты и времени в машиночитаемый формат. 
            
          <p>
            Бронирование действительно на <time datetime="20:00">20:00</time>
          </p>
  • Атрибут даты ISO 8601(datetime): используется для представления даты и времени в машиночитаемом формате. Стандартный формат — ГГГГ-ММ-ДДTчч:мм:сс, где заглавная буква T служит разделителем между датой и временем.
  • Элемент надстрочный индекс(sup): используется для представления надстрочного текста. Элемент sup обычно используется для обозначения показателей степеней, высших букв и порядковых числительных. 
            
          <p>
            CO<sub>2</sub>
          </p>
  • Элемент Встроенный код(code): используется для представления фрагмента компьютерного кода.
  • Элемент Предварительно отформатированный текст(pre): представляет собой предварительно отформатированный текст. 
    •       
        <pre>
          <code>
            body {
              color: red;
            }
          </code>
        </pre>
  • Элемент Неартикулированная аннотация(u): используется для представления фрагмента встроенного текста, который должен быть отображен таким образом, чтобы было понятно, что он содержит нетекстовую аннотацию. 
            
          <p>
            Вы можете использовать элемент неартикулированной аннотации, чтобы выделить
            <u>несооответствие</u> <u>нописание</u> <u>проблеммы</u>.
          </p>
  • Элемент Ruby-аннотация(ruby): используется для аннотации текста с пояснениями произношения или значения. Пример использования — восточноазиатская типографика.
  • Элемент Резервные скобки Ruby(rp): используется как запасной вариант для браузеров, не поддерживающих отображение аннотаций Ruby.
  • Элемент текста Ruby(rt): используется для обозначения текста аннотации Ruby. Обычно используется для обозначения произношения или деталей перевода в восточноазиатской типографике. 
    •       
        <ruby>
          明日 <rp>(</rp><rt>Ashita</rt><rp>)</rp>
        </ruby>
  • Элемент Зачеркивание(s): используется для обозначения содержимого, которое больше не является точным или актуальным. 
            
          <p><s>Завтрашний поход состоится в полдень.</s></p>

          <p>Из-за непредвиденных погодных условий поход был отменен.</p>
