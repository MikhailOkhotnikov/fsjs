Что такое атрибут aria-hidden и как он работает?

Если вам когда-либо понадобится отображать контент, одновременно скрывая его от пользователей вспомогательных технологий, таких как программы чтения с экрана, вы можете использовать атрибут aria-hidden.

Вам просто нужно добавить его к HTML-элементу, который вы хотите скрыть, и установить его значение равным true, как показано здесь: aria-hidden="true".

Этот атрибут скрывает элемент и все его дочерние элементы из дерева доступности, сохраняя при этом их видимыми на странице. Типичные варианты использования:

  • Иконки и изображения, имеющие исключительно декоративное назначение.
  • Дублированный контент.

Важно помнить, что атрибут aria-hidden скрывает контент только от вспомогательных технологий, таких как программы чтения с экрана. Если контент должен быть скрыт от всех, то не следует использовать aria-hidden для его скрытия. Например, меню-гамбургер(hamburger menu), которое в данный момент свернуто, должно быть скрыто от всех пользователей клавиатуры, а не только от пользователей программ чтения с экрана. В этом случае можно установить свойство CSS display в значение none для меню, чтобы удалить его из DOM, когда оно свернуто.

Никогда не следует использовать атрибут aria-hidden для скрытия элемента, который можно сфокусировать с помощью клавиатуры. Атрибут aria-hidden только удаляет элемент из дерева доступности. Он не удаляет его из DOM. Таким образом, пользователи программ чтения с экрана по-прежнему смогут перейти к элементу с помощью клавиши Tab, но их программа чтения с экрана не будет озвучивать этот элемент, фактически заставляя их фокусироваться на «ничём».

Использовать атрибут aria-hidden не нужно, если:

  • HTML-элемент уже имеет атрибут hidden.
  • Этот элемент или его предок уже скрыт с помощью CSS свойства display: none.
  • Этот элемент или его предок уже скрыт с помощью CSS свойства visibility: hidden.

В этих трех случаях элемент уже удален из DOM и, следовательно, скрыт от дерева доступности, поэтому атрибут aria-hidden не требуется.

Как и при использовании любых атрибутов ARIA, всегда следует проводить тестирование с помощью вспомогательных технологий, и желательно, чтобы вашу работу тестировали люди с ограниченными возможностями, чтобы убедиться в ее простоте понимания, даже с учетом скрытых элементов.

Также следует знать, что установка атрибута aria-hidden в значение false не позволит вспомогательным технологиям использовать этот элемент, даже если у какого-либо из его родительских элементов этот атрибут установлен в значение true.

Атрибут aria-hidden используется для скрытия элементов от пользователей вспомогательных технологий, таких как программы чтения с экрана.

Хотя это может быть полезно для скрытия чисто декоративных элементов и дублирующегося контента, его следует использовать экономно, чтобы не препятствовать доступности.

В целом, весь контент и функциональность, доступные на странице, должны быть доступны и пользователям вспомогательных технологий. Применение aria-hidden очень узкое и должно ограничиваться в первую очередь улучшением пользовательского опыта для пользователей программ чтения с экрана путем удаления чисто декоративной или дублирующейся информации. Не используйте aria-hidden для скрытия контента, который, по вашему мнению, не заинтересует пользователей программ чтения с экрана. Пользователи программ чтения с экрана заслуживают доступа ко всей информации на странице.

Следуя этим передовым практикам и тестируя пользовательский опыт, вы можете создать инклюзивные онлайн-ресурсы для всех.
