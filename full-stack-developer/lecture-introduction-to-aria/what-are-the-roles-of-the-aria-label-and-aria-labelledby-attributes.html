Каковы функции атрибутов aria-label и aria-labeledby?

Важно обеспечить беспрепятственный доступ к веб-сайтам для всех пользователей, включая людей с ограниченными возможностями.

Для пользователей программ чтения с экрана атрибуты aria-label и aria-labelledby предоставляют важную информацию об элементах страницы, которые могут быть нечеткими или невидимыми.

Давайте рассмотрим, что представляют собой атрибуты aria-label и aria-labelledby и какую роль они играют в обеспечении доступности веб-сайтов для людей с нарушениями зрения и связанными с ними ограничениями.

Вы заметите, что атрибуты aria-label и aria-labelledby имеют префикс aria. Что это значит? ARIA расшифровывается как Accessible Rich Internet Applications (доступные многофункциональные интернет-приложения). Это набор атрибутов с префиксом aria-, который позволяет разработчикам сообщать вспомогательным технологиям о назначении элементов. Атрибут aria-label — это невидимая метка для интерактивных элементов. Он добавляет текстовую метку к элементу, которую могут прочитать программы чтения с экрана.

aria-label особенно полезен для элементов, не содержащих видимого текста, но которые тем не менее нуждаются в описании для программ чтения с экрана. Например, кнопки, состоящие только из значков, часто нуждаются в aria-label для передачи своего назначения.

Вот пример: 

    
      <button aria-label="Search">
        <i class="fa-solid fa-magnifying-glass"></i>
      </button>

В этом случае программа чтения с экрана может объявить эту кнопку как Search, button(Поиск, кнопка), даже если кнопка содержит только значок. Атрибут aria-label указывает программам чтения с экрана, какой текст использовать вместо значка.

Если бы кнопка содержала текст «Поиск» вместо значка, то атрибут aria-label не понадобился бы, поскольку текст сам по себе служил бы меткой для кнопки.

Для элементов ввода атрибут aria-label напрямую предоставляет метку, если с элементом ввода не связана видимая метка.

Атрибут aria-labelledby делает то же самое, что и атрибут aria-label, но вместо того, чтобы определять текст непосредственно в атрибуте, используется ссылка на текст, который уже существует на странице. Существующий текст должен иметь атрибут id, который будет использоваться в качестве значения ссылки в атрибуте aria-labelledby.

Вот пример: 

    
      <input type="text" aria-labelledby="search-btn">
      <button type="button" id="search-btn">Поиск</button>

В данном случае текст кнопки используется в качестве метки для поля поиска. Программы чтения с экрана озвучат поле ввода примерно так: Поиск, редактировать. Если позже вы решите изменить текст кнопки на Найти, метка поля ввода автоматически обновится до нового текста, поскольку она ссылается на текст кнопки. Также возможно объединение нескольких значений id в одно значение атрибута aria-labelledby. Вот как это работает: 

    
      <div>
        <span id="volume-label">Громкость</span>
        <span id="volume-details">Отрегулируйте уровень громкости</span>
        <input
          type="range"
          min="0"
          max="100"
          value="30"
          aria-labelledby="volume-label volume-details">
      </div>

Для ползунка программа чтения с экрана будет отслеживать содержимое элементов volume-label и volume-details и объявлять Громкость. Отрегулируйте уровень громкости..

Вы уже видели, что атрибуты aria-label и aria-labelledby помогают программам чтения с экрана понимать, что делают элементы. Так какой же из них следует использовать? Поскольку они оба предоставляют одинаковую функциональность, можно использовать любой из них, но у aria-labelledby могут быть некоторые преимущества перед aria-label:

  • Если кто-то использует сервис перевода для перевода контента на вашей странице, текст в атрибуте aria-label может быть переведен не всегда.
  • Использование атрибута aria-labelledby также может помочь предотвратить несоответствие между видимым и невидимым текстом метки для пользователей программ чтения с экрана, поскольку обновление видимого текста автоматически обновит невидимую метку.
  • aria-labelledby значительно упрощает программное создание сложных невидимых меток, состоящих из нескольких источников текста.

И последнее замечание: не используйте одновременно атрибуты aria-label и aria-labelledby для одного элемента. В этом случае невидимая метка для программ чтения с экрана всегда будет определяться атрибутом aria-labelledby, а атрибут aria-label будет полностью игнорироваться.
