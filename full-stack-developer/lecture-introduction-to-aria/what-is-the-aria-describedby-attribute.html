Что такое атрибут aria-describedby и как он работает?

Атрибут aria-describedby используется для предоставления дополнительной информации об элементе пользователям программ чтения с экрана путем ссылки на существующий контент на странице. Он создает программную связь между элементом и контентом (технически называемую доступным описанием), которую программы чтения с экрана могут использовать для информирования пользователей о дополнительной информации при взаимодействии с элементом.

Наиболее распространенное применение атрибута aria-describedby — это связывание инструкций и сообщений об ошибках с полями ввода формы. Из-за различных способов навигации по странице, используемых пользователями программ чтения с экрана, они могут пропустить эти сообщения при переходе между полями ввода. Использование aria-describedby помогает гарантировать, что они их услышат.

Давайте рассмотрим несколько примеров, чтобы понять, как это работает. В первом примере у нас есть элемент form, который принимает пароль.

Включите интерактивный редактор и введите несколько символов в поле ввода пароля. Вы увидите, что пароль скрыт в окне предварительного просмотра. Также вы заметите, что текст справки по паролю остается красным, пока вы не введете 8 или более символов в поле ввода.

ПРИМЕЧАНИЕ: В этом интерактивном примере для динамического обновления цвета текста password-help используются CSS и JavaScript. Не беспокойтесь о понимании кода JavaScript, поскольку вы изучите его в последующих модулях.

Файл index.html 
      
        <link rel="stylesheet" href="styles.css">

        <form>
          <label for="password">Пароль:</label>
          <input type="password" id="password" aria-describedby="password-help" />
          <p id="password-help">Ваш пароль должен состоять как минимум из 8 символов.</p>
        </form>

        <script src="index.js"></script>
Файл styles.css 
      
        #password-help {
          color: red;
        }
Файл index.js 
      
        const passwordEl = document.getElementById("password");
        const passwordHelpText = document.getElementById("password-help");

        passwordEl.addEventListener("input", (e) => { 
          const userInput = e.target.value;
          passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
        });

Мы используем элемент label для текста Пароль: и связываем его с input пароля.

У нас также есть элемент абзаца, описывающий требования к паролю. Мы используем атрибут aria-describedby, чтобы связать поле ввода пароля с требованиями к паролю в элементе абзаца. Когда пользователь программы чтения с экрана взаимодействует с этим полем ввода, его программа чтения с экрана озвучит имя поля ввода (Пароль:), а затем может также озвучить требования к паролю. Однако это не является абсолютной гарантией, поскольку некоторые программы чтения с экрана могут не озвучивать дополнительное содержимое автоматически или озвучивать его только в определенных обстоятельствах. Это не редкость. Каждая программа чтения с экрана работает и обрабатывает атрибуты ARIA по-своему. Это не отменяет использования aria-describedby, поскольку это принесет пользу пользователям программ чтения с экрана в целом.

Ещё один удачный пример использования атрибута aria-describedby — это кнопка удаления. Вот пример button удаления, за которой следует сообщение, описывающее, что произойдёт при нажатии на кнопку:

Файл index.html 
      
        <button aria-describedby="delete-message">Удалить</button>

        <p id="delete-message">Внимание! Все удаления необратимы.</p>

Как и в предыдущем примере, мы связываем кнопку удаления с сообщением, используя атрибут aria-describedby. Значение id и значение атрибута aria-describedby должны совпадать.

Атрибут aria-describedby — это мощный атрибут, который можно использовать для обеспечения предоставления дополнительной информации об элементе пользователям программ чтения с экрана при взаимодействии с ним. Чаще всего он используется для связывания инструкций и сообщений об ошибках с полями ввода формы, чтобы уменьшить вероятность того, что пользователи программ чтения с экрана пропустят эти сообщения при навигации по форме.
