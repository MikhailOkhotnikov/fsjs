В чём цель WAI-ARIA и как она работает?

Обеспечение доступности статического контента может быть относительно простым делом, но с динамическим контентом могут возникнуть более сложные задачи. Именно здесь на помощь приходит WAI-ARIA.

Давайте рассмотрим, что такое WAI-ARIA, каково её назначение, как она работает и приведём несколько примеров.

WAI-ARIA расшифровывается как Web Accessibility Initiative - Accessible Rich Internet Applications (Инициатива по обеспечению доступности веб-сайтов — доступные многофункциональные интернет-приложения). Это спецификация, повышающая доступность динамического контента и компонентов пользовательского интерфейса (UI, User Interface).

Следует отметить, что WCAG и WAI-ARIA — это не одно и то же. WCAG предоставляет общие рекомендации по доступности веб-сайтов, в то время как WAI-ARIA предлагает конкретные правила для обеспечения доступности динамического и интерактивного контента для пользователей вспомогательных технологий.

Таким образом, основная цель WAI-ARIA — улучшить доступность динамического контента и компонентов пользовательского интерфейса, для которых отсутствуют собственные HTML-эквиваленты.

WAI-ARIA работает за счет введения набора атрибутов, которые можно добавлять к HTML-элементам для предоставления дополнительной семантической информации. Эти атрибуты подразделяются на роли, состояния и свойства.

Роль ARIA определяет назначение элемента на веб-сайте или в веб-приложении. Вот пример установки роли для элемента div в значение button. 

    
      <div role="button">Нажмите сюда</div>

Таким образом, вы сообщаете вспомогательным технологиям, что элемент является кнопкой. Однако роли не предоставляют никакой функциональности. Простое присвоение div role="button" не заставит его вести себя как кнопка. Чтобы он выглядел и вел себя как кнопка, вам потребуется использовать CSS и JavaScript для достижения желаемого результата.

Вот пример использования HTML, CSS и JavaScript для создания пользовательского элемента button.

Примечание: Не беспокойтесь о том, чтобы пытаться понять код CSS и JavaScript. Вы изучите эти языки в последующих модулях.

Файл index.html 
      
        <link href="styles.css" rel="stylesheet">
        <div id="custom-btn" role="button">Нажмите сюда</div>
        <script src="index.js"></script>
Файл styles.css 
      
        #custom-btn {
          display: inline-block;
          padding: 0.4em 1em;
          font-size: 1rem;
          font-family: sans-serif;
          color: buttontext;
          background-color: buttonface;
          border: 1px solid #ccc;
          border-radius: 4px;
          cursor: pointer;
          user-select: none;
          text-align: center;
        }

        #custom-btn:focus {
          outline: 2px solid Highlight;
          outline-offset: 2px;
        }

        #custom-btn:active {
          background-color: #ddd;
        }
Файл index.js 
      
        const button = document.getElementById("custom-btn");

        button.addEventListener("click", () => {
          alert("Button clicked!");
        });

        button.addEventListener("keydown", (e) => {
          if (e.key === "Enter" || e.key === " ") {
            e.preventDefault(); 
            button.click();
          }
        });

Всегда лучше использовать стандартную button или input с атрибутом type="button".

Свойства ARIA предоставляют дополнительные сведения об элементах. Например, свойство aria-labelledby позволяет связать элемент с определенной меткой: 

    
      <h2 id="header-id">О freeCodeCamp</h2>
      <button id="button-id" aria-labelledby="header-id button-id">Узнать больше</button>

Это сделает элементы понятными и удобными для пользователей вспомогательных технологий.

Чтобы максимально эффективно использовать WAI-ARIA, старайтесь по возможности использовать нативный HTML, поскольку он, как правило, обеспечивает лучшую доступность «из коробки».

Используйте WAI-ARIA только тогда, когда HTML не справляется, и не забывайте тестировать с помощью вспомогательных технологий, таких как программы чтения с экрана, или привлекайте к тестированию людей с ограниченными возможностями. Также убедитесь, что состояния и свойства WAI-ARIA обновляются в реальном времени вместе с содержимым. Избегайте чрезмерного использования ARIA, так как это часто может вызывать путаницу.
