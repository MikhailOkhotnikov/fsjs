Обзор основ CSS

Основы CSS

  • Что такое CSS?: Каскадные таблицы стилей (Cascading Style Sheets, CSS) — это язык разметки, используемый для применения стилей к элементам HTML. CSS используется для цветов, фоновых изображений, макетов и многого другого.
  • Базовая структура правила CSS: Правило CSS состоит из двух основных частей: селектора и блока объявления. Селектор — это шаблон, используемый в CSS для идентификации и выбора конкретных HTML-элементов для стилизации. Блок объявления применяет набор стилей к заданному селектору или селекторам.

    Вот общий синтаксис правила CSS: 

            
          selector {
              property: value;
          }
  • Элемент meta name="viewport": Этот meta элемент дает браузеру инструкции по управлению размерами и масштабированием страницы на различных устройствах, в частности на мобильных телефонах и планшетах.
  • Стили браузера по умолчанию: К каждому HTML-элементу применяются стили браузера по умолчанию. Обычно это включает такие параметры, как внутренние(padding) и внешние(margin) отступы по умолчанию.

Встроенные, внутренние и внешние CSS-стили

  • Встроенные CSS-стили (Inline CSS): эти стили пишутся непосредственно внутри HTML-элемента с помощью атрибута style. В большинстве случаев вы не будете использовать встроенные CSS-стили из-за принципа разделения ответственности.

    Вот пример встроенного CSS: 

            
          <link rel="stylesheet" href="styles.css">
          <p style="color: red;">Это абзац, выделенный красным цветом.</p>
  • Внутренние CSS-стили (Internal CSS): Эти стили записываются внутри тегов style в разделе head HTML-документа. Это может быть полезно для создания коротких примеров кода, но обычно вам не потребуется использовать внутренние CSS-стили.
  • Внешние CSS-стили (External CSS): Эти стили пишутся в отдельном CSS-файле и связываются с HTML-документом с помощью элемента link в разделе head. В большинстве проектов вы будете использовать внешний CSS-файл вместо внутреннего или встроенного CSS.

Работа со свойствами ширины и высоты

  • Свойство width: это свойство задает ширину элемента. Если ширина не указана, по умолчанию устанавливается значение auto. Это позволяет браузеру определять ширину элемента на основе его содержимого, родительского элемента и типа отображения.
  • Свойство min-width: Это свойство задает минимальную ширину элемента.
  • Свойство max-width: Это свойство задает максимальную ширину элемента.
  • Свойство height: Это свойство задает высоту элемента. Аналогично, по умолчанию высота устанавливается автоматически (auto), что означает, что она будет подстраиваться под содержимое элемента.
  • Свойство min-height: Это свойство задает минимальную высоту элемента.
  • Свойство max-height: Это свойство задает максимальную высоту элемента.

Различные типы комбинаторов CSS

  • Комбинатор потомков (Descendant Combinator): Этот комбинатор используется для выбора элементов, являющихся потомками указанного родительского элемента. В следующем примере будут выбраны все элементы li внутри элементов ul.
    • Файл index.html 
              
            <link rel="stylesheet" href="styles.css">
            <ul>
                <li>Пример первого пункта</li>
                <li>Пример второго пункта</li>
                <li>Пример третьего пункта</li>
            </ul>
    Файл styles.css 
              
            ul li {
                background-color: yellow;
            }
  • Комбинатор дочерних элементов (Child Combinator, >): Этот комбинатор используется для выбора элементов, являющихся непосредственными потомками указанного родительского элемента. В следующем примере будут выбраны все элементы p, являющиеся непосредственными потомками класса container.
    • Файл index.html 
              
            <link rel="stylesheet" href="styles.css">
            <div class="container">
              <p>Это будет оформлено.</p>

              <div>
                <p>Это не будет оформлено.</p>
              </div>
            </div>
    Файл styles.css 
              
            .container > p {
              background-color: black;
              color: white;
            }
  • Комбинатор "Следующий соседний элемент" (Next-sibling Combinator, +): этот комбинатор выбирает элемент, который следует сразу за указанным одноуровневым элементом. В следующем примере будет выбран элемент p, который следует сразу за элементом h2.
    • Файл index.html 
              
            <link rel="stylesheet" href="styles.css">
            <h2>Я являюсь подзаголовком</h2>

            <p>Этот элемент абзаца получит красный фон.</p>
    Файл styles.css 
              
            h2 + p {
              background-color: red;
            }
  • Комбинатор "последующие соседние элементы" (Subsequent-sibling Combinator, ~): Этот комбинатор выбирает все соседние элементы указанного элемента, которые следуют за ним. В следующем примере будет применен стиль только ко второму элементу абзаца, поскольку он является единственным соседним элементом элемента ul и имеет того же родителя.
    • Файл index.html 
              
            <link rel="stylesheet" href="styles.css">
            <div class="container">
              <p>Это не будет оформлено.</p>
              <ul>
                <li>Пример первого пункта</li>
                <li>Пример второго пункта</li>
                <li>Пример третьего пункта</li>
              </ul>
              <p>Это будет оформлено.</p>
            </div>
            <p>Это не будет оформлено.</p>
    Файл styles.css 
              
            ul ~ p {
              background-color: green;
            }

Элементы строчного, блочного и строчно-блочного уровней

  • Элементы строчного уровня (Inline Level Elements): Встроенные (или строчные) элементы занимают только необходимую ширину и не начинаются с новой строки. Эти элементы располагаются внутри контента, позволяя тексту и другим встроенным элементам отображаться рядом с ними. К распространенным строчным (встроенным) элементам относятся span, a и img.
  • Элементы блочного уровня (Block Level Elements): Блочные элементы начинаются с новой строки и по умолчанию занимают всю доступную им ширину, растягиваясь на всю ширину своего контейнера. К распространенным блочным элементам относятся элементы div, p и section.
  • Элементы строчно-блочного уровня (Inline-Block Level Elements): Вы можете присвоить элементу статус inline-block, используя свойство display. Эти элементы ведут себя как строчные элементы, но могут иметь ширину (width) и высоту (height), заданные как блочные элементы.

Внешние (margin) и внутренние (padding) отступы

  • Свойство margin: Это свойство используется для добавления пространства за пределами элемента, между границей элемента и окружающими элементами.
  • Свойство padding: Это свойство используется для добавления пространства внутри элемента, между содержимым и его границей.
  • Сокращенная запись margin: Вы можете указать от 1 до 4 значений для задания внешних отступов поля. Одно значение применяется ко всем четырем сторонам; два значения задают верхнюю (top) и нижнюю (bottom) стороны, затем правую (right) и левую (left); три значения задают верхнюю (top), горизонтальную (справа (right) и слева (left)), затем нижнюю (bottom) сторону; четыре значения задают верхнюю (top), правую (right), нижнюю (bottom) и левую (left) стороны.
  • Сокращенная запись padding: Вы можете указать от 1 до 4 значений для задания внутренних отступов поля. Одно значение применяется ко всем четырем сторонам; два значения задают верхнюю (top) и нижнюю (bottom) стороны, затем правую (right) и левую (left); три значения задают верхнюю (top), горизонтальную (справа (right) и слева (left)), затем нижнюю (bottom) сторону; четыре значения задают верхнюю (top), правую (right), нижнюю (bottom) и левую (left) стороны.

Специфичность CSS

  • Специфичность встроенных CSS-стилей (Inline CSS Specificity): Встроенные CSS-стили обладают наивысшей специфичностью, поскольку применяются непосредственно к элементу. Они переопределяют любые внутренние или внешние CSS-стили. Значение специфичности для встроенных стилей равно (1, 0, 0, 0).
  • Специфичность внутренних CSS-стилей (Internal CSS Specificity): Внутренние CSS-стили определяются внутри элемента style в разделе head HTML-документа. Они имеют более низкую специфичность, чем встроенные стили. Как внутренние, так и внешние CSS-стили имеют одинаковый уровень специфичности, который определяется их селекторами, а не их местоположением.
  • Специфичность внешних CSS-стилей (External CSS Specificity): Внешние CSS-стили связаны через элемент link в разделе head и написаны в отдельных файлах .css. Как и внутренние CSS-стили, их специфичность определяется используемыми селекторами. Когда два правила имеют одинаковую специфичность, порядок в исходном коде определяет победителя — правило, которое появляется позже в документе, имеет приоритет. Внешние CSS-стили обеспечивают наилучшую поддерживаемость для крупных проектов.
  • Универсальный селектор (Universal Selector, *): особый тип CSS-селектора, который соответствует любому элементу в документе. Он часто используется для применения стиля ко всем элементам на странице, что может быть полезно для сброса или нормализации стилей в разных браузерах. Универсальный селектор имеет наименьшее значение специфичности среди всех селекторов. Он вносит 0 во все части значения специфичности (0, 0, 0, 0).
  • Селекторы типов (Type Selectors): Эти селекторы выбирают элементы на основе их имени тега. Селекторы типов имеют относительно низкую специфичность по сравнению с другими селекторами. Значение специфичности для селектора типа равно (0, 0, 0, 1).
  • Селекторы классов (Class Selectors): Эти селекторы определяются точкой (.), за которой следует имя класса. Значение специфичности для селектора класса равно (0, 0, 1, 0). Это означает, что селекторы классов могут переопределять селекторы типов, но могут быть переопределены селекторами идентификаторов и встроенными стилями.
  • Селекторы идентификаторов (ID Selectors): Селекторы идентификаторов определяются символом решетки (#), за которым следует имя ID. Селекторы идентификаторов имеют очень высокую специфичность, выше, чем у селекторов типов и селекторов классов, но ниже, чем у встроенных стилей. Значение специфичности для селектора идентификаторов равно (0, 1, 0, 0).
  • Ключевое слово !important: используется для присвоения правилу стиля наивысшего приоритета, позволяя ему переопределять любые другие объявления для свойства. При использовании оно заставляет браузер применять указанный стиль независимо от специфичности других селекторов. Следует проявлять осторожность при использовании !important, поскольку это может затруднить поддержку и отладку вашего CSS-кода.
  • Каскадный алгоритм (Cascade Algorithm): алгоритм, используемый для определения того, какие правила CSS применять, когда к одному и тому же элементу применяется несколько стилей. Он гарантирует использование наиболее подходящих стилей на основе набора четко определенных правил.
  • Наследование CSS (CSS Inheritance): процесс передачи стилей от родительских элементов к дочерним. Наследование позволяет определять стили на более высоком уровне в дереве документа и применять их к нескольким элементам без явного указания для каждого элемента отдельно.
